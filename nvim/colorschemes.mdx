## Overview

This configuration includes seven beautiful colorschemes with OneDark set as the default. All themes are installed and ready to use.

## Current Active Theme

**OneDark** is set as the default colorscheme with a custom visual highlight configuration.

## Configuration

All colorschemes are configured in `lua/plugins/colorscheme.lua`:

```lua
return {
  { "rebelot/kanagawa.nvim" },
  { "NLKNguyen/papercolor-theme" },
  { "ellisonleao/gruvbox.nvim" },
  { "rose-pine/neovim" },
  { "tiagovla/tokyodark.nvim" },
  { "sainnhe/everforest" },
  {
    "navarasu/onedark.nvim",
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "onedark",
    },
  },
}
```

## Switching Colorschemes

### Temporarily (Current Session)

Use the `:colorscheme` command:

```vim
:colorscheme kanagawa
:colorscheme gruvbox
:colorscheme rose-pine
```

Or use LazyVim's colorscheme picker:

```
<leader>uC
```

### Permanently (Save to Config)

Edit `lua/plugins/colorscheme.lua` and change the colorscheme setting:

```lua
{
  "LazyVim/LazyVim",
  opts = {
    colorscheme = "kanagawa",  -- Change to your preferred theme
  },
}
```

Restart Neovim or run `:LazyReload` to apply changes.

## Visual Selection Highlight

The configuration includes a custom visual selection highlight that applies globally:

**In `lua/config/autocmds.lua`:**
```lua
vim.api.nvim_set_hl(0, "Visual", { bg = "#ffff00", fg = "#000000" })
```

**In `lua/plugins/colorscheme.lua` (OneDark specific):**
```lua
opts = {
  highlights = {
    Visual = { bg = "#ffff00" },
  },
}
```

This creates a bright yellow visual selection that overrides the theme's default visual highlight for maximum visibility.

<Note>
  The yellow visual selection is set in two places: globally in autocmds.lua and specifically for OneDark. You may want to adjust or remove this if you prefer theme-native visual highlights.
</Note>

## Adding More Colorschemes

To add a new colorscheme:

1. Add the plugin to `lua/plugins/colorscheme.lua`:

```lua
return {
  -- ... existing colorschemes ...
  { "username/new-colorscheme.nvim" },
}
```

2. Restart Neovim or run `:Lazy sync`

3. Activate with `:colorscheme new-colorscheme`
