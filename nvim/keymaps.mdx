## Custom Keymaps

This configuration primarily relies on LazyVim's default keybindings. The `lua/config/keymaps.lua` file is present for custom keybindings but currently contains no custom mappings:

```lua title="lua/config/keymaps.lua"
-- Keymaps are automatically loaded on the VeryLazy event
-- Default keymaps that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua
-- Add any additional keymaps here
```

Custom completion behavior is configured in `tab-complete.lua`:

| Key | Mode | Description |
|-----|------|-------------|
| `<Tab>` | i, s | Select first item or confirm selection |
| `<CR>` | i, s, c | Dismiss completion menu (normal Enter behavior) |
| `<C-n>` | i | Next completion item |
| `<C-p>` | i | Previous completion item |

- `:ObsidianSearch` - Search notes
- `:ObsidianQuickSwitch` - Quick switch between notes
- `:ObsidianLink` - Create link to note
- `:ObsidianBacklinks` - Show backlinks

## Customizing Keymaps

To add your own keymaps, edit `lua/config/keymaps.lua`. LazyVim uses `vim.keymap.set()` for defining keybindings:

```lua
-- Example custom keymap
vim.keymap.set("n", "<leader>xx", function()
  -- Your custom function
end, { desc = "Description" })
```

**Best practices:**
- Use descriptive `desc` fields - they appear in which-key
- Avoid overriding LazyVim defaults unless necessary
- Group related keymaps under the same leader prefix
- Use `<leader>` for custom commands to avoid conflicts

## Which-Key

LazyVim includes [which-key.nvim](https://github.com/folke/which-key.nvim), which displays available keybindings in a popup. Press `<leader>` and wait briefly to see all available shortcuts.

You can also search keymaps:
- `:Telescope keymaps` - Browse all keymaps with Telescope
- `:WhichKey` - Show all which-key mappings
