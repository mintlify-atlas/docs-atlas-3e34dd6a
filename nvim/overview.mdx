## Introduction

This is a modern Neovim configuration built on top of [LazyVim](https://www.lazyvim.org/), a powerful and extensible Neovim setup. The configuration follows LazyVim's conventions while adding custom plugins and settings tailored for productivity and development workflows.

## Architecture

### Bootstrap Process

The configuration starts with a minimal `init.lua` that bootstraps the entire setup:

```lua title="init.lua"
-- bootstrap lazy.nvim, LazyVim and your plugins
require("config.lazy")
```

### Directory Structure

The configuration is organized into the following structure:

```
nvim/
├── init.lua                    # Entry point
├── lazyvim.json               # LazyVim extras configuration
├── lua/
│   ├── config/
│   │   ├── lazy.lua           # lazy.nvim bootstrap and setup
│   │   ├── options.lua        # Custom vim options
│   │   ├── keymaps.lua        # Custom keybindings
│   │   └── autocmds.lua       # Custom autocommands
│   └── plugins/               # Plugin specifications
│       ├── colorscheme.lua
│       ├── leetcode.lua
│       ├── obsidian.lua
│       ├── snacks-config.lua
│       ├── tab-complete.lua
│       ├── vimbegood.lua
│       └── mason-workaround.lua
```

### lazy.nvim Setup

The configuration uses [lazy.nvim](https://github.com/folke/lazy.nvim) as the plugin manager. The setup in `lua/config/lazy.lua` automatically clones lazy.nvim if it's not present and configures it to load both LazyVim's default plugins and custom plugins:

```lua title="lua/config/lazy.lua"
require("lazy").setup({
  spec = {
    -- add LazyVim and import its plugins
    { "LazyVim/LazyVim", import = "lazyvim.plugins" },
    -- import/override with your plugins
    { import = "plugins" },
  },
  defaults = {
    lazy = false,
    version = false, -- always use the latest git commit
  },
  install = { colorscheme = { "tokyonight", "habamax" } },
  checker = {
    enabled = true, -- check for plugin updates periodically
    notify = false, -- notify on update
  },
})
```

## Key Features

### Multiple Colorschemes

The configuration includes 7 colorschemes with OneDark as the default, featuring custom visual selection highlighting (yellow background via autocmd).

### Language Support

LazyVim extras provide enhanced support for multiple languages and frameworks:
- **AI Tools**: GitHub Copilot Chat integration
- **Coding**: Enhanced nvim-cmp completion
- **Languages**: C/C++ (clangd), Go, JSON, Python, Tailwind CSS, TypeScript

### Productivity Tools

- **LeetCode Integration**: Practice coding problems directly in Neovim
- **Obsidian Integration**: Seamless note-taking with your Obsidian vault
- **Vim Be Good**: Practice and improve your Vim motions

### Enhanced Completion

- Custom Tab completion behavior that auto-selects the first item
- Enter key to dismiss completion without inserting
- Emoji support in completion menu

### File Picker

Snacks.nvim picker configured to show hidden files and gitignored files.

### Stable Tooling

Mason versions are pinned to ensure consistent LSP and tooling behavior.

## Getting Started

This configuration is designed to work out of the box. Simply clone it to your Neovim configuration directory:

```bash
git clone <repository-url> ~/.config/nvim
```

On first launch, lazy.nvim will automatically install all plugins. LazyVim will also install required LSP servers, formatters, and linters through Mason.

## Philosophy

This configuration embraces LazyVim's philosophy of:
- Sensible defaults that work for most developers
- Easy customization through plugin specs
- Lazy loading for fast startup times
- Regular updates to stay current with the ecosystem

Custom plugins and settings are added only when they provide clear value over the defaults, keeping the configuration maintainable and focused.
