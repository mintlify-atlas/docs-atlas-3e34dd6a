## Configuration File

Alacritty is configured using a `alacritty.toml` file located at `~/.config/alacritty/alacritty.toml`. The configuration uses TOML format for clear, type-safe settings.

## Complete Configuration

Here's the complete configuration from this dotfiles setup:

```toml alacritty.toml
general.import = [
    "~/.config/alacritty/themes/themes/coolnight.toml"
]
[ env ]
TERM = "xterm-256color"

[window]
padding.x = 10
padding.y = 10

decorations = "Buttonless"
opacity = 0.8
blur = true
startup_mode = "Maximized"
[font]
normal.family = "0xProto Nerd Font Mono"

#regular
size = 16
```

## Configuration Sections

### General

```toml
general.import = [
    "~/.config/alacritty/themes/themes/coolnight.toml"
]
```

The `general.import` directive loads external configuration files, primarily used for theme management. This allows you to:
- Keep theme configuration separate from main settings
- Easily switch between different color schemes
- Share themes across multiple machines

<Note>
Import paths can use `~` for home directory expansion. Theme files are imported before the main configuration is processed.
</Note>

### Environment Variables

```toml
[env]
TERM = "xterm-256color"
```

Setting `TERM` to `xterm-256color` ensures:
- Full 256-color support in terminal applications
- Proper compatibility with programs like vim, tmux, and htop
- Correct terminal capabilities detection

### Window Settings

```toml
[window]
padding.x = 10
padding.y = 10
decorations = "Buttonless"
opacity = 0.8
blur = true
startup_mode = "Maximized"
```

#### Padding

- **padding.x**: Horizontal padding (10 pixels) adds space on left and right
- **padding.y**: Vertical padding (10 pixels) adds space on top and bottom

This creates breathing room between terminal content and window edges.

#### Decorations

```toml
decorations = "Buttonless"
```

Removes standard window buttons (minimize, maximize, close) for a cleaner, more minimal appearance. Options include:
- `"Full"`: Standard window decorations
- `"None"`: No title bar or buttons
- `"Transparent"`: Transparent title bar
- `"Buttonless"`: Title bar without buttons

#### Opacity and Blur

```toml
opacity = 0.8
blur = true
```

- **opacity**: Sets window transparency (0.0 = fully transparent, 1.0 = fully opaque)
- **blur**: Applies background blur effect when opacity < 1.0

<Note>
Background blur requires compositor support (e.g., picom on Linux, built-in on macOS). Without a compositor, the background will be transparent but not blurred.
</Note>

#### Startup Mode

```toml
startup_mode = "Maximized"
```

Launches Alacritty in maximized mode by default. Other options:
- `"Windowed"`: Normal window size
- `"Maximized"`: Fill screen (excluding dock/taskbar)
- `"Fullscreen"`: True fullscreen mode

### Font Configuration

```toml
[font]
normal.family = "0xProto Nerd Font Mono"
size = 16
```

#### Font Family

```toml
normal.family = "0xProto Nerd Font Mono"
```

Uses [0xProto Nerd Font](https://github.com/ryanoasis/nerd-fonts), which includes:
- Programming ligatures for better code readability
- Extensive icon glyphs (devicons, font awesome, etc.)
- Powerline symbols for status lines
- Box drawing characters

<Note>
Make sure the font is installed system-wide. Nerd Fonts can be installed via:
- Homebrew: `brew install font-0xproto-nerd-font`
- Manual download from [Nerd Fonts releases](https://github.com/ryanoasis/nerd-fonts/releases)
</Note>

#### Font Size

```toml
size = 16
```

Sets the font size to 16 points. Adjust based on:
- Screen resolution and DPI
- Personal preference
- Display size

## Modifying Configuration

To modify the configuration:

<Steps>
  <Step title="Edit the file">
    Open `~/.config/alacritty/alacritty.toml` in your editor
  </Step>
  <Step title="Make changes">
    Update the desired settings using TOML syntax
  </Step>
  <Step title="Save and test">
    Alacritty automatically reloads configuration when the file changes
  </Step>
</Steps>

<Note>
Alacritty watches the configuration file and hot-reloads most settings. If changes don't apply, restart Alacritty or check for syntax errors in the terminal output.
</Note>
