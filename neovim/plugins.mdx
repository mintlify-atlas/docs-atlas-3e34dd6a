## Plugin Management

All custom plugins are defined as Lua modules in the `lua/plugins/` directory. Each file returns a table (or array of tables) describing plugin specifications for lazy.nvim.

<Note>
  In addition to these custom plugins, LazyVim provides dozens of pre-configured plugins for LSP, treesitter, debugging, and more. These custom plugins extend or override the base LazyVim setup.
</Note>

## Colorscheme Plugins

The configuration includes multiple colorscheme options, with OneDark set as the default (nvim/lua/plugins/colorscheme.lua:1-24).

```lua title="nvim/lua/plugins/colorscheme.lua"
return {
  { "rebelot/kanagawa.nvim" },
  { "NLKNguyen/papercolor-theme" },
  { "ellisonleao/gruvbox.nvim" },
  { "rose-pine/neovim" },
  { "tiagovla/tokyodark.nvim" },
  { "sainnhe/everforest" },
  {
    "navarasu/onedark.nvim",
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "onedark",
    },
  },
}
```

### Theme Customization

The OneDark theme includes a custom highlight override (nvim/lua/plugins/colorscheme.lua:11-16):

```lua
opts = {
  highlights = {
    Visual = { bg = "#ffff00" },
  },
}
```

This changes the visual selection background to bright yellow (`#ffff00`) for better visibility.

<Tip>
  To change themes, modify the `colorscheme` value in the LazyVim opts (line 20). Run `:Lazy sync` to apply changes.
</Tip>

## Obsidian Integration

The Obsidian plugin enables seamless integration with Obsidian vaults directly from Neovim (nvim/lua/plugins/obsidian.lua:1-36).

```lua title="nvim/lua/plugins/obsidian.lua"
return {
  "epwalsh/obsidian.nvim",
  version = "*",
  lazy = true,
  ft = "markdown",
  dependencies = {
    "nvim-lua/plenary.nvim",
    "hrsh7th/nvim-cmp",
  },
  opts = {
    workspaces = {
      {
        name = "personal",
        path = "/Users/leylandyang/Library/Mobile Documents/iCloud~md~obsidian/Documents/Sharedvault",
      },
    },
    completion = {
      nvim_cmp = true,
      min_chars = 2,
    },
    new_notes_location = "notes_subdir",
    daily_notes = {
      folder = "/daily notes",
      date_format = "%m-%d-%Y",
      alias_format = "%B %-d, %Y",
      default_tags = { "daily-notes" },
      template = nil,
    },
  },
}
```

### Configuration Details

<Tabs>
  <Tab title="Loading">
    The plugin uses lazy loading and only activates when opening markdown files (nvim/lua/plugins/obsidian.lua:4-5):
    ```lua
    lazy = true,
    ft = "markdown",
    ```
    This improves startup time by not loading Obsidian functionality until needed.
  </Tab>
  
  <Tab title="Workspace">
    Configured to work with an iCloud-synced Obsidian vault (nvim/lua/plugins/obsidian.lua:11-16):
    ```lua
    workspaces = {
      {
        name = "personal",
        path = "/Users/leylandyang/Library/Mobile Documents/iCloud~md~obsidian/Documents/Sharedvault",
      },
    }
    ```
    <Note>
      You'll need to update the path to match your own Obsidian vault location.
    </Note>
  </Tab>
  
  <Tab title="Completion">
    Integrates with nvim-cmp for wiki-link and tag completion (nvim/lua/plugins/obsidian.lua:17-20):
    ```lua
    completion = {
      nvim_cmp = true,
      min_chars = 2,
    }
    ```
    Start typing `[[` or `#` to trigger completion after 2 characters.
  </Tab>
  
  <Tab title="Daily Notes">
    Configured for daily note creation (nvim/lua/plugins/obsidian.lua:22-33):
    ```lua
    daily_notes = {
      folder = "/daily notes",
      date_format = "%m-%d-%Y",
      alias_format = "%B %-d, %Y",
      default_tags = { "daily-notes" },
    }
    ```
    Creates daily notes in the format `01-15-2026.md` with title "January 15, 2026".
  </Tab>
</Tabs>

## Snacks.nvim Configuration

Customizes the Snacks.nvim picker to show hidden and ignored files (nvim/lua/plugins/snacks-config.lua:1-12).

```lua title="nvim/lua/plugins/snacks-config.lua"
return {
  {
    "folke/snacks.nvim",
    opts = {
      picker = {
        hidden = true,
        ignored = true,
      },
    },
  },
}
```

<Note>
  This configuration enables searching through hidden files (starting with `.`) and files listed in `.gitignore` when using the Snacks picker.
</Note>

## Completion Behavior

Customizes nvim-cmp completion behavior for a smoother experience (nvim/lua/plugins/tab-complete.lua:1-40).

```lua title="nvim/lua/plugins/tab-complete.lua"
return {
  {
    "L3MON4D3/LuaSnip",
    keys = function()
      return {}
    end,
  },
  {
    "hrsh7th/nvim-cmp",
    dependencies = {
      "hrsh7th/cmp-emoji",
    },
    opts = function(_, opts)
      local cmp = require("cmp")
      
      opts.mapping = vim.tbl_extend("force", opts.mapping, {
        ["<Tab>"] = cmp.mapping(function(fallback)
          if cmp.visible() then
            local entry = cmp.get_selected_entry()
            if not entry then
              cmp.select_next_item({ behavior = cmp.SelectBehavior.Select })
            else
              cmp.confirm()
            end
          else
            fallback()
          end
        end, { "i", "s" }),
        ["<CR>" = cmp.mapping(function(fallback)
          cmp.abort()
          fallback()
        end, { "i", "s", "c" }),
      })
    end,
  },
}
```

### Customized Behavior

<Tabs>
  <Tab title="Tab Key">
    Smart Tab completion (nvim/lua/plugins/tab-complete.lua:17-30):
    
    - If completion menu is visible with selection: **confirm** the selected item
    - If completion menu is visible without selection: **select** the first item
    - If no completion menu: perform normal Tab action
    
    This provides a smooth "Tab to complete" workflow.
  </Tab>
  
  <Tab title="Enter Key">
    Modified Enter behavior (nvim/lua/plugins/tab-complete.lua:31-34):
    
    ```lua
    ["<CR>"] = cmp.mapping(function(fallback)
      cmp.abort()
      fallback()
    end, { "i", "s", "c" }),
    ```
    
    Enter always creates a new line instead of confirming completion. Use Tab to accept suggestions.
  </Tab>
  
  <Tab title="LuaSnip">
    Disables LuaSnip keybindings (nvim/lua/plugins/tab-complete.lua:3-6):
    
    ```lua
    "L3MON4D3/LuaSnip",
    keys = function()
      return {}
    end,
    ```
    
    This prevents conflicts with the custom Tab completion behavior.
  </Tab>
  
  <Tab title="Emoji Support">
    Adds emoji completion (nvim/lua/plugins/tab-complete.lua:11):
    
    ```lua
    dependencies = {
      "hrsh7th/cmp-emoji",
    },
    ```

  </Tab>
</Tabs>
