## Overview

This guide will walk you through installing and configuring a complete development environment on macOS with Neovim, Alacritty, Tmux, and AeroSpace.

<Note>
  This setup is designed specifically for macOS. Some components (like AeroSpace) are macOS-only.
</Note>

## Prerequisites

<Steps>
  <Step title="Verify macOS">
    Ensure you're running macOS 10.15 (Catalina) or later:
    ```bash
    sw_vers
    ```
  </Step>

  <Step title="Install Homebrew">
    If you don't have Homebrew installed, install it first:
    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```
  </Step>

  <Step title="Install Xcode Command Line Tools">
    Required for building certain packages:
    ```bash
    xcode-select --install
    ```
  </Step>
</Steps>

## Installing Base Tools

<Steps>
  <Step title="Install Core Applications">
    Install all required applications using Homebrew:
    ```bash
    brew install neovim tmux alacritty
    brew install --cask nikitabobko/tap/aerospace
    ```
  </Step>

  <Step title="Install 0xProto Nerd Font">
    Install the Nerd Font used in these configurations:
    ```bash
    brew tap homebrew/cask-fonts
    brew install --cask font-0xproto-nerd-font
    ```
  </Step>

  <Step title="Install Tmux Plugin Manager (TPM)">
    TPM manages Tmux plugins:
    ```bash
    git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
    ```
  </Step>
</Steps>

## Cloning the Configuration

<Steps>
  <Step title="Clone the dotfiles repository">
    Clone this repository to your preferred location:
    ```bash
    git clone https://github.com/yangleyland/configs ~/dotfiles
    cd ~/dotfiles
    ```
  </Step>

  <Step title="Backup existing configurations">
    If you have existing configurations, back them up:
    ```bash
    # Backup existing configs
    mv ~/.config/nvim ~/.config/nvim.backup 2>/dev/null || true
    mv ~/.config/alacritty ~/.config/alacritty.backup 2>/dev/null || true
    mv ~/.tmux.conf ~/.tmux.conf.backup 2>/dev/null || true
    mv ~/.aerospace.toml ~/.aerospace.toml.backup 2>/dev/null || true
    ```
  </Step>
</Steps>

## Symlinking Configurations

<Steps>
  <Step title="Create symlinks for Neovim">
    Link the Neovim configuration:
    ```bash
    ln -sf ~/dotfiles/nvim ~/.config/nvim
    ```
  </Step>

  <Step title="Create symlinks for Alacritty">
    Link the Alacritty configuration:
    ```bash
    ln -sf ~/dotfiles/alacritty ~/.config/alacritty
    ```
  </Step>

  <Step title="Create symlinks for Tmux">
    Link the Tmux configuration:
    ```bash
    ln -sf ~/dotfiles/tmux/tmux.conf ~/.tmux.conf
    ```
  </Step>

  <Step title="Create symlinks for AeroSpace">
    Link the AeroSpace configuration:
    ```bash
    ln -sf ~/dotfiles/aerospace/aerospace.toml ~/.aerospace.toml
    ```
  </Step>
</Steps>

<Warning>
  The symlink paths assume you cloned the repository to `~/dotfiles`. Adjust paths accordingly if you used a different location.
</Warning>

## Initial Setup

<Steps>
  <Step title="Launch Neovim and install plugins">
    Open Neovim for the first time. LazyVim will automatically install all plugins:
    ```bash
    nvim
    ```
    Wait for all plugins to download and install. This may take a few minutes.
  </Step>

  <Step title="Install Tmux plugins">
    Launch Tmux and install plugins:
    ```bash
    tmux
    ```
    Inside Tmux, press `Ctrl+Space` then `I` (capital i) to install plugins.
  </Step>

  <Step title="Configure AeroSpace to start at login">
    The configuration is already set to `start-at-login = true`. Start AeroSpace:
    ```bash
    open -a AeroSpace
    ```
    Grant necessary accessibility permissions when prompted in System Preferences.
  </Step>

  <Step title="Configure Alacritty as default terminal (optional)">
    To set Alacritty as your default terminal:
    - Open System Preferences > General > Default Terminal
    - Or use it as your primary terminal application
  </Step>
</Steps>

## Verification

Verify your installation:

<CodeGroup>
```bash Neovim
# Check Neovim version (should be 0.9.0+)
nvim --version

# Check LazyVim health
nvim +checkhealth
```

```bash Tmux
# Check Tmux version
tmux -V

# List installed plugins
ls ~/.tmux/plugins/
```

```bash Alacritty
# Check Alacritty version
alacritty --version

# Test font rendering
alacritty -e echo "Testing 0xProto Nerd Font: "
```

```bash AeroSpace
# Check AeroSpace version
aerospace --version

# List current workspaces
aerospace list-workspaces
```
</CodeGroup>

## Post-Installation Configuration

<Steps>
  <Step title="Customize Neovim colorscheme">
    The default colorscheme is OneDark. To change it, edit:
    ```
    ~/.config/nvim/lua/plugins/colorscheme.lua
    ```
    Available themes: Kanagawa, Gruvbox, Rose Pine, Tokyo Dark, Everforest, OneDark
  </Step>

  <Step title="Customize Alacritty theme">
    The default theme is CoolNight. To change it, edit:
    ```
    ~/.config/alacritty/alacritty.toml
    ```
    Browse available themes in `~/.config/alacritty/themes/themes/`
  </Step>

  <Step title="Adjust AeroSpace keybindings">
    If the default Alt-based keybindings conflict with your workflow:
    ```
    ~/.aerospace.toml
    ```
    Modify the `[mode.main.binding]` section
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Neovim plugins not loading">
    Try removing the lazy-lock and reinstalling:
    ```bash
    rm ~/.config/nvim/lazy-lock.json
    nvim
    ```
    Then run `:Lazy sync` inside Neovim.
  </Accordion>

  <Accordion title="Tmux plugins not installing">
    Ensure TPM is installed correctly:
    ```bash
    ls -la ~/.tmux/plugins/tpm
    ```
    If missing, reinstall TPM:
    ```bash
    git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
    ```
  </Accordion>

  <Accordion title="Font not displaying correctly">
    Verify the Nerd Font is installed:
    ```bash
    ls ~/Library/Fonts | grep 0xProto
    ```
    If missing, reinstall:
    ```bash
    brew reinstall --cask font-0xproto-nerd-font
    ```
  </Accordion>

  <Accordion title="AeroSpace not starting">
    Check if you've granted accessibility permissions:
    - System Preferences > Privacy & Security > Accessibility
    - Ensure AeroSpace is checked
    - Restart AeroSpace after granting permissions
  </Accordion>

  <Accordion title="Alacritty transparency not working">
    Ensure you're not using a non-compositing window manager and that transparency is enabled in macOS:
    - Check System Preferences > Accessibility > Display
    - Ensure "Reduce transparency" is OFF
  </Accordion>
</AccordionGroup>

<Note>
  After installation, you may want to restart your terminal sessions to ensure all changes take effect.
</Note>
