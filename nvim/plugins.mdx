## Overview

This configuration extends LazyVim with carefully selected plugins for enhanced productivity, aesthetics, and workflow integration. All plugins are managed through lazy.nvim and are defined in the `lua/plugins/` directory.

## LazyVim Extras

LazyVim extras provide pre-configured plugin bundles for specific languages and features. This configuration includes the following extras (defined in `lazyvim.json`):

**AI Tools:**
- `lazyvim.plugins.extras.ai.copilot-chat` - GitHub Copilot Chat integration

**Coding:**
- `lazyvim.plugins.extras.coding.nvim-cmp` - Enhanced completion with nvim-cmp

**Language Support:**
- `lazyvim.plugins.extras.lang.clangd` - C/C++ LSP support via clangd
- `lazyvim.plugins.extras.lang.go` - Go development tools
- `lazyvim.plugins.extras.lang.json` - JSON support and formatting
- `lazyvim.plugins.extras.lang.python` - Python development tools
- `lazyvim.plugins.extras.lang.tailwind` - Tailwind CSS integration
- `lazyvim.plugins.extras.lang.typescript` - TypeScript/JavaScript development

These extras install and configure LSP servers, formatters, linters, and additional plugins specific to each language or feature.

Seamless integration with Obsidian vaults for note-taking and knowledge management.

```lua title="lua/plugins/obsidian.lua"
return {
  "epwalsh/obsidian.nvim",
  version = "*",
  lazy = true,
  ft = "markdown",
  dependencies = {
    "nvim-lua/plenary.nvim",
    "hrsh7th/nvim-cmp",
  },
  opts = {
    workspaces = {
      {
        name = "personal",
        path = "/Users/leylandyang/Library/Mobile Documents/iCloud~md~obsidian/Documents/Sharedvault",
      },
    },
    completion = {
      nvim_cmp = true,
      min_chars = 2,
    },
    new_notes_location = "notes_subdir",
    daily_notes = {
      folder = "/daily notes",
      date_format = "%m-%d-%Y",
      alias_format = "%B %-d, %Y",
      default_tags = { "daily-notes" },
      template = nil,
    },
  },
}
```

**Features:**
- Full Obsidian vault integration
- Note linking and backlinks
- Daily notes with customizable format
- nvim-cmp completion for note names and tags
- Lazy loads only for markdown files

**Configuration details:**
- Workspace path configured for iCloud-synced Obsidian vault
- Daily notes use MM-DD-YYYY format
- Notes are created in subdirectories by default
- Auto-tagging for daily notes

**Repository:** [epwalsh/obsidian.nvim](https://github.com/epwalsh/obsidian.nvim)

## Completion

### Tab Complete

Custom completion behavior for a more intuitive experience.

<Accordion title="Tab Complete Configuration">

```lua title="lua/plugins/tab-complete.lua"
return {
  {
    "L3MON4D3/LuaSnip",
    keys = function()
      return {}
    end,
  },
  {
    "hrsh7th/nvim-cmp",
    dependencies = {
      "hrsh7th/cmp-emoji",
    },
    opts = function(_, opts)
      local cmp = require("cmp")

      opts.mapping = vim.tbl_extend("force", opts.mapping, {
        ["<Tab>"] = cmp.mapping(function(fallback)
          -- This little snippet will confirm with tab, and if no entry is selected,
          -- will confirm the first item
          if cmp.visible() then
            local entry = cmp.get_selected_entry()
            if not entry then
              cmp.select_next_item({ behavior = cmp.SelectBehavior.Select })
            else
              cmp.confirm()
            end
          else
            fallback()
          end
        end, { "i", "s" }),
        ["<CR>"] = cmp.mapping(function(fallback)
          cmp.abort()
          fallback()
        end, { "i", "s", "c" }),
      })
    end,
  },
}
```

**Features:**
- Tab automatically selects and confirms the first completion item if none is selected
- Enter key dismisses completion without inserting (normal newline behavior)
- Emoji completion support
- LuaSnip key mappings disabled to prevent conflicts

**Behavior:**
1. Press Tab when completion menu is visible → selects first item if none selected, or confirms selected item
2. Press Enter → closes completion menu and inserts newline
3. Use emoji trigger (`:`) to get emoji completions

**Reference:** Configuration based on [LazyVim discussion #250](https://github.com/LazyVim/LazyVim/discussions/250)

</Accordion>

## Plugin Loading Strategy

Most custom plugins follow LazyVim's conventions:

- **Lazy loading**: Plugins like Obsidian (`ft = "markdown"`) and Vim Be Good (`cmd = "VimBeGood"`) only load when needed
- **Eager loading**: Core plugins and colorschemes load at startup
- **Dependency management**: All dependencies are explicitly declared
- **Version pinning**: Obsidian uses stable releases (`version = "*"`), while Mason versions are pinned for stability

This approach ensures fast startup times while maintaining full functionality.
