## Keymap Configuration

Custom keymaps are defined in `lua/config/keymaps.lua`. This configuration follows LazyVim's convention of loading keymaps automatically on the VeryLazy event.

```lua title="nvim/lua/config/keymaps.lua"
-- Keymaps are automatically loaded on the VeryLazy event
-- Default keymaps that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua
-- Add any additional keymaps here
```

<Note>
  Currently, this configuration file doesn't define any custom keymaps beyond LazyVim defaults. All keybindings come from LazyVim's base configuration.
</Note>

## Plugin-Specific Keymaps

Some plugins define their own keybindings:

### Obsidian.nvim

When editing markdown files in your Obsidian vault:

| Keymap | Command | Action |
|--------|---------|--------|
| `gf` | (built-in) | Follow link under cursor |
| - | `:ObsidianNew` | Create new note |
| - | `:ObsidianToday` | Open today's daily note |
| - | `:ObsidianSearch` | Search notes |
| - | `:ObsidianQuickSwitch` | Quick note switcher |

<Tip>
  You can create custom keymaps for Obsidian commands in `lua/config/keymaps.lua`. For example:
  ```lua
  vim.keymap.set("n", "<leader>on", "<cmd>ObsidianNew<cr>", { desc = "New Obsidian note" })
  vim.keymap.set("n", "<leader>ot", "<cmd>ObsidianToday<cr>", { desc = "Today's note" })
  ```
</Tip>

### LeetCode.nvim

Access LeetCode functionality:

| Command | Action |
|---------|--------|
| `:Leet` | Open LeetCode menu |
| `:Leet list` | List problems |
| `:Leet desc` | Show problem description |
| `:Leet run` | Run test cases |
| `:Leet submit` | Submit solution |

## Adding Custom Keymaps

To add your own keybindings, edit `lua/config/keymaps.lua`:

```lua title="nvim/lua/config/keymaps.lua"
-- Example: Add a keymap to save file with Ctrl+S
vim.keymap.set({ "i", "n", "v" }, "<C-s>", "<cmd>w<cr><esc>", { desc = "Save file" })

-- Example: Quick access to config
vim.keymap.set("n", "<leader>fc", function()
  require("telescope.builtin").find_files({ cwd = vim.fn.stdpath("config") })
end, { desc = "Find config files" })

-- Example: Toggle diagnostics
vim.keymap.set("n", "<leader>ud", function()
  vim.diagnostic.enable(not vim.diagnostic.is_enabled())
end, { desc = "Toggle diagnostics" })
```

<Note>
  Always include a `desc` (description) field in your keymaps. This makes them searchable via `<leader>sk` (search keymaps).
</Note>

## Keymap Reference

For a complete list of LazyVim's default keymaps, visit:

- [LazyVim Keymaps Documentation](https://www.lazyvim.org/keymaps)
- [LazyVim Source - keymaps.lua](https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua)

You can also search keymaps interactively in Neovim:
- Press `<leader>sk` to search all keymaps
- Press `<leader>?` to see keymaps for current buffer
