## Overview

This Neovim configuration relies primarily on LazyVim's comprehensive default keybindings. Custom keymaps are minimal to maintain compatibility with LazyVim's well-designed defaults.

## Custom Keymaps

The `lua/config/keymaps.lua` file currently contains no custom keybindings:

```lua
-- Keymaps are automatically loaded on the VeryLazy event
-- Default keymaps that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua
-- Add any additional keymaps here
```

This means the configuration uses 100% of LazyVim's default keybindings without modifications.

## Plugin-Specific Keybindings

Some plugins add their own keybindings:

### LeetCode.nvim

Access LeetCode commands via command mode or through which-key menu.

### Vim Be Good

Start the practice game with:
```
:VimBeGood
```

### Obsidian.nvim

Obsidian commands are available via the `:Obsidian` command prefix when editing markdown files.

## Which-Key Integration

Press `<leader>` and wait briefly to see a popup menu showing all available keybindings. This is powered by which-key.nvim and makes discovering keybindings easy.

## Adding Custom Keymaps

To add custom keymaps, edit `lua/config/keymaps.lua`:

```lua
local map = vim.keymap.set

-- Example: Map <leader>h to toggle search highlight
map("n", "<leader>h", ":set hlsearch!<CR>", { desc = "Toggle search highlight" })
```

<Note>
  Always add a `desc` (description) field to your keymaps so they show up properly in which-key.
</Note>
