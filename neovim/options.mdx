## Options Configuration

Editor options are configured in `lua/config/options.lua`. This file is automatically loaded before lazy.nvim startup, ensuring options are set early in the initialization process.

```lua title="nvim/lua/config/options.lua"
-- Options are automatically loaded before lazy.nvim startup
-- Default options that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/options.lua
-- Add any additional options here
-- vim.g.snacks_animate = false
vim.g.root_spec = { "cwd" }
```

<Note>
  This configuration file is minimal, containing only one custom option. Most editor settings come from LazyVim's sensible defaults.
</Note>

## Custom Options

### Root Directory Detection

The only custom option overrides LazyVim's root directory detection (nvim/lua/config/options.lua:5):

```lua
vim.g.root_spec = { "cwd" }
```

<Tabs>
  <Tab title="What It Does">
    This setting changes how Neovim determines the "root" directory of your project.
    
    **Default LazyVim behavior:** Looks for markers like `.git`, `package.json`, etc. to detect project root
    
    **Custom behavior:** Always uses the current working directory (`cwd`) as the root
  </Tab>
  
  <Tab title="Why Use It">
    Benefits of using `cwd` as root:
    
    - More predictable behavior
    - No automatic directory changes
    - Better for multi-project workflows
    - Simpler mental model
    
    <Tip>
      If you work with nested Git repositories or monorepos, you might prefer LazyVim's default root detection. Comment out this line to restore default behavior.
    </Tip>
  </Tab>
  
  <Tab title="Impact">
    This affects several features:
    
    - **File searching** - Telescope searches from cwd
    - **LSP** - Language servers use cwd as workspace root
    - **Git operations** - Git commands run from cwd
    - **Terminal** - New terminals open in cwd
  </Tab>
</Tabs>

### Animation Setting (Commented)

The file includes a commented-out option (nvim/lua/config/options.lua:4):

```lua
-- vim.g.snacks_animate = false
```

<Accordion title="About Snacks Animations">
  **Snacks.nvim** includes animations for various UI elements (scrolling, notifications, etc.).
  
  If you find animations distracting or experience performance issues:
  
  ```lua
  vim.g.snacks_animate = false  -- Disable all animations
  ```
  
  Currently, this is commented out, meaning animations are **enabled** by default.
</Accordion>

## LazyVim Default Options

While not explicitly defined in your config, LazyVim sets many sensible defaults. Here are the most important ones:

### Editor Behavior

<AccordionGroup>
  <Accordion title="General Settings">
    ```lua
    vim.opt.autowrite = true           -- Auto write before commands
    vim.opt.clipboard = "unnamedplus"  -- Sync with system clipboard
    vim.opt.completeopt = "menu,menuone,noselect"
    vim.opt.conceallevel = 2           -- Hide markup characters
    vim.opt.confirm = true             -- Confirm to save changes
    vim.opt.formatoptions = "jcroqlnt" -- Format options
    vim.opt.grepformat = "%f:%l:%c:%m"
    vim.opt.grepprg = "rg --vimgrep"   -- Use ripgrep for grep
    vim.opt.ignorecase = true          -- Ignore case in search
    vim.opt.smartcase = true           -- Don't ignore case with capitals
    vim.opt.inccommand = "nosplit"     -- Preview incremental substitute
    vim.opt.laststatus = 3             -- Global statusline
    vim.opt.mouse = "a"                -- Enable mouse support
    vim.opt.pumblend = 10              -- Popup transparency
    vim.opt.pumheight = 10             -- Maximum popup items
    vim.opt.sessionoptions = { "buffers", "curdir", "tabpages", "winsize", "help", "globals", "skiprtp", "folds" }
    vim.opt.shortmess:append({ W = true, I = true, c = true, C = true })
    vim.opt.showmode = false           -- Don't show mode (in statusline)
    vim.opt.sidescrolloff = 8          -- Columns of context
    vim.opt.signcolumn = "yes"         -- Always show sign column
    vim.opt.smartindent = true         -- Smart autoindenting
    vim.opt.spelllang = { "en" }
    vim.opt.splitbelow = true          -- New horizontal splits below
    vim.opt.splitkeep = "screen"       -- Keep screen position on split
    vim.opt.splitright = true          -- New vertical splits right
    vim.opt.termguicolors = true       -- True color support
    vim.opt.timeoutlen = 300           -- Timeout for key sequences
    vim.opt.undofile = true            -- Persistent undo
    vim.opt.undolevels = 10000
    vim.opt.updatetime = 200           -- Faster CursorHold events
    vim.opt.virtualedit = "block"      -- Virtual editing in visual block
    vim.opt.wildmode = "longest:full,full" -- Command-line completion
    vim.opt.winminwidth = 5            -- Minimum window width
    vim.opt.wrap = false               -- Disable line wrap
    ```
  </Accordion>
  
  <Accordion title="Indentation">
    ```lua
    vim.opt.expandtab = true           -- Use spaces instead of tabs
    vim.opt.shiftround = true          -- Round indent to multiple of shiftwidth
    vim.opt.shiftwidth = 2             -- Size of an indent
    vim.opt.tabstop = 2                -- Number of spaces tabs count for
    vim.opt.softtabstop = 2
    ```
    
    <Note>
      LazyVim uses **2 spaces** for indentation by default. This works well for most languages but can be overridden per-filetype.
    </Note>
  </Accordion>
  
  <Accordion title="Line Numbers">
    ```lua
    vim.opt.number = true              -- Print line numbers
    vim.opt.relativenumber = true      -- Relative line numbers
    ```
    
    This combination shows:
    - Current line: absolute number
    - Other lines: relative distance
    
    Perfect for motion commands like `5j` (move 5 lines down).
  </Accordion>
  
  <Accordion title="Scrolling">
    ```lua
    vim.opt.scrolloff = 4              -- Lines of context around cursor
    vim.opt.sidescrolloff = 8          -- Columns of context
    vim.opt.smoothscroll = true        -- Smooth scrolling
    ```
    
    Keeps cursor away from screen edges for better context.
  </Accordion>
  
  <Accordion title="Folding">
    ```lua
    vim.opt.foldmethod = "expr"        -- Use expression for folds
    vim.opt.foldexpr = "v:lua.vim.treesitter.foldexpr()"
    vim.opt.foldlevel = 99             -- Open all folds by default
    vim.opt.foldlevelstart = 99
    vim.opt.foldenable = true
    ```
    
    Uses TreeSitter for intelligent code folding.
  </Accordion>
  
  <Accordion title="Backup & Undo">
    ```lua
    vim.opt.backup = false
    vim.opt.writebackup = false
    vim.opt.swapfile = false
    vim.opt.undofile = true            -- Persistent undo
    vim.opt.undodir = vim.fn.stdpath("state") .. "/undo"
    ```
    
    Disables swap files but enables persistent undo history across sessions.
  </Accordion>
</AccordionGroup>

## Global Variables

LazyVim also sets several global variables that affect plugin behavior:

```lua
vim.g.mapleader = " "              -- Set leader key to space
vim.g.maplocalleader = "\\"        -- Set local leader to backslash
vim.g.lazyvim_picker = "auto"     -- Auto-detect picker (telescope/fzf)
```

## Overriding Options

To customize any option, add it to `lua/config/options.lua`:

### Example Overrides

<CodeGroup>
```lua title="Indentation (4 spaces)"
-- Use 4 spaces instead of 2
vim.opt.shiftwidth = 4
vim.opt.tabstop = 4
vim.opt.softtabstop = 4
```

```lua title="Enable Line Wrap"
-- Enable line wrapping
vim.opt.wrap = true
vim.opt.linebreak = true  -- Break at word boundaries
```

```lua title="Disable Mouse"
-- Disable mouse support
vim.opt.mouse = ""
```

```lua title="Absolute Line Numbers"
-- Use only absolute line numbers
vim.opt.number = true
vim.opt.relativenumber = false
```

```lua title="Faster Key Timeout"
-- Reduce timeout for key sequences
vim.opt.timeoutlen = 200  -- Default is 300
```

```lua title="More Scrolloff"
-- More context around cursor
vim.opt.scrolloff = 8
vim.opt.sidescrolloff = 15
```
</CodeGroup>

## Filetype-Specific Options

You can set options for specific filetypes using autocmds:

```lua title="lua/config/options.lua"
-- Example: 4-space indent for Python
vim.api.nvim_create_autocmd("FileType", {
  pattern = "python",
  callback = function()
    vim.opt_local.shiftwidth = 4
    vim.opt_local.tabstop = 4
    vim.opt_local.softtabstop = 4
  end,
})

-- Example: Enable spell check for markdown
vim.api.nvim_create_autocmd("FileType", {
  pattern = { "markdown", "text" },
  callback = function()
    vim.opt_local.spell = true
    vim.opt_local.wrap = true
    vim.opt_local.linebreak = true
  end,
})

-- Example: Tabs for Go
vim.api.nvim_create_autocmd("FileType", {
  pattern = "go",
  callback = function()
    vim.opt_local.expandtab = false  -- Use real tabs
    vim.opt_local.tabstop = 4
  end,
})
```

<Tip>
  Use `vim.opt_local` instead of `vim.opt` in FileType autocmds to avoid affecting other buffers.
</Tip>

## Viewing Current Options

To check the current value of any option in Neovim:

```vim
:set <option>?      " View single option
:set all            " View all options
:verbose set <option>?  " See where option was last set
```

Or in Lua:

```lua
:lua print(vim.inspect(vim.opt.shiftwidth:get()))
```
