This guide shows you how to customize the visual appearance of your development environment by changing color schemes across all tools.

## Overview

The Desktop Configs setup uses coordinated themes across:
- **Neovim**: Code editor color schemes
- **Alacritty**: Terminal emulator colors
- **Tmux**: Terminal multiplexer styling

<Tip>
For a cohesive look, choose matching or complementary themes across all three tools. Popular choices include Catppuccin, Gruvbox, and Tokyo Night variants.
</Tip>

## Neovim Color Schemes

### Available Themes

The configuration comes with several pre-installed color schemes defined in `nvim/lua/plugins/colorscheme.lua`:

<CodeGroup>
```lua nvim/lua/plugins/colorscheme.lua
return {
  { "rebelot/kanagawa.nvim" },
  { "NLKNguyen/papercolor-theme" },
  { "ellisonleao/gruvbox.nvim" },
  { "rose-pine/neovim" },
  { "tiagovla/tokyodark.nvim" },
  { "sainnhe/everforest" },
  {
    "navarasu/onedark.nvim",
    opts = {
      highlights = {
        Visual = { bg = "#ffff00" },
      },
    },
  },
  {
    "LazyVim/LazyVim",
    opts = {
      colorscheme = "onedark",
    },
  },
}
```
</CodeGroup>

### Change the Active Theme

<Steps>
  <Step title="Edit the colorscheme.lua file">
    Open `~/.config/nvim/lua/plugins/colorscheme.lua` in your editor
  </Step>
  
  <Step title="Update the colorscheme option">
    Find the `LazyVim` configuration block and change the `colorscheme` value:
    
    ```lua
    {
      "LazyVim/LazyVim",
      opts = {
        colorscheme = "gruvbox",  -- Change this to your preferred theme
      },
    }
    ```
  </Step>
  
  <Step title="Restart Neovim">
    Close and reopen Neovim to apply the changes. The new theme will be loaded automatically.
  </Step>
</Steps>

### Add New Color Schemes

To install additional themes not included by default:

<Steps>
  <Step title="Add the plugin to colorscheme.lua">
    Add the theme's plugin specification to the return table:
    
    ```lua
    return {
      -- Existing themes...
      { "folke/tokyonight.nvim" },  -- Add new theme
      
      {
        "LazyVim/LazyVim",
        opts = {
          colorscheme = "tokyonight",
        },
      },
    }
    ```
  </Step>
  
  <Step title="Lazy.nvim will auto-install">
    When you restart Neovim, Lazy.nvim will automatically download and install the new plugin.
  </Step>
</Steps>

### Customize Theme Highlights

You can override specific highlight groups to fine-tune colors:

```lua
{
  "navarasu/onedark.nvim",
  opts = {
    highlights = {
      Visual = { bg = "#ffff00" },        -- Yellow visual selection
      Comment = { fg = "#7c8696", italic = true },
      LineNr = { fg = "#4b5263" },
    },
  },
}
```

<Note>
Refer to each theme's documentation for available configuration options and customization possibilities.
</Note>

## Alacritty Terminal Theming

### Current Theme Setup

Alacritty themes are configured in `alacritty/alacritty.toml`:

```toml alacritty/alacritty.toml
general.import = [
    "~/.config/alacritty/themes/themes/coolnight.toml"
]

[window]
opacity = 0.8
blur = true
```

### Available Themes

The configuration includes many pre-downloaded themes in the `alacritty/themes/themes/` directory, including:
- Cobalt2
- Mariana
- Gruvbox variants
- Tokyo Night variants
- Nord
- And many more

### Switch Alacritty Themes

<Steps>
  <Step title="Browse available themes">
    List all available theme files:
    
    ```bash
    ls ~/.config/alacritty/themes/themes/
    ```
  </Step>
  
  <Step title="Edit alacritty.toml">
    Open `~/.config/alacritty/alacritty.toml` and update the import path:
    
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/gruvbox_dark.toml"
    ]
    ```
  </Step>
  
  <Step title="Save and see changes">
    Alacritty automatically reloads configuration changes. Your terminal colors will update immediately.
  </Step>
</Steps>

### Create Custom Theme

You can create your own Alacritty theme:

<Steps>
  <Step title="Create a new theme file">
    ```bash
    touch ~/.config/alacritty/themes/themes/mytheme.toml
    ```
  </Step>
  
  <Step title="Define color scheme">
    ```toml mytheme.toml
    [colors.primary]
    background = '#1e1e2e'
    foreground = '#cdd6f4'
    
    [colors.normal]
    black   = '#45475a'
    red     = '#f38ba8'
    green   = '#a6e3a1'
    yellow  = '#f9e2af'
    blue    = '#89b4fa'
    magenta = '#f5c2e7'
    cyan    = '#94e2d5'
    white   = '#bac2de'
    
    [colors.bright]
    black   = '#585b70'
    red     = '#f38ba8'
    green   = '#a6e3a1'
    yellow  = '#f9e2af'
    blue    = '#89b4fa'
    magenta = '#f5c2e7'
    cyan    = '#94e2d5'
    white   = '#a6adc8'
    ```
  </Step>
  
  <Step title="Import your theme">
    Update `alacritty.toml` to use your custom theme:
    
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/mytheme.toml"
    ]
    ```
  </Step>
</Steps>

<Tip>
You can use an existing theme file as a template. Copy it and modify the hex color values to create your custom palette.
</Tip>

## Tmux Status Bar Theming

### Current Theme

The Tmux configuration uses the Catppuccin theme plugin:

```bash tmux/tmux.conf
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'dreamsofcode-io/catppuccin-tmux'
```

### Change Tmux Theme

<Steps>
  <Step title="Edit tmux.conf">
    Open `~/.config/tmux/tmux.conf`
  </Step>
  
  <Step title="Replace the theme plugin">
    Comment out the current theme and add a new one:
    
    ```bash
    set -g @plugin 'tmux-plugins/tpm'
    # set -g @plugin 'dreamsofcode-io/catppuccin-tmux'
    set -g @plugin 'jimeh/tmux-themepack'
    set -g @themepack 'powerline/default/cyan'
    ```
    
    Popular theme plugins:
    - `jimeh/tmux-themepack`
    - `catppuccin/tmux`
    - `dracula/tmux`
    - `arcticicestudio/nord-tmux`
  </Step>
  
  <Step title="Install the theme">
    Press `Ctrl+Space` + `I` (capital i) inside Tmux to install the new plugin.
  </Step>
  
  <Step title="Reload configuration">
    ```bash
    tmux source ~/.config/tmux/tmux.conf
    ```
  </Step>
</Steps>

### Custom Status Bar Colors

For manual status bar customization without plugins:

```bash
# Status bar colors
set -g status-style 'bg=#1e1e2e fg=#cdd6f4'
set -g status-left-style 'bg=#89b4fa fg=#1e1e2e'
set -g status-right-style 'bg=#f38ba8 fg=#1e1e2e'

# Window status colors
set -g window-status-current-style 'bg=#a6e3a1 fg=#1e1e2e'
set -g window-status-style 'bg=#45475a fg=#cdd6f4'

# Pane border colors
set -g pane-border-style 'fg=#45475a'
set -g pane-active-border-style 'fg=#89b4fa'
```

## Coordinated Theme Recommendations

Here are some well-matched theme combinations:

<Accordion title="Catppuccin Mocha">
  - **Neovim**: Install `catppuccin/nvim` and set colorscheme to `catppuccin-mocha`
  - **Alacritty**: Use `catppuccin-mocha.toml`
  - **Tmux**: Use `catppuccin/tmux` with mocha flavor
</Accordion>

<Accordion title="Gruvbox Dark">
  - **Neovim**: Already included, set colorscheme to `gruvbox`
  - **Alacritty**: Use `gruvbox_dark.toml`
  - **Tmux**: Use `egel/tmux-gruvbox` plugin
</Accordion>

<Accordion title="Tokyo Night">
  - **Neovim**: Install `folke/tokyonight.nvim`
  - **Alacritty**: Use `tokyo-night.toml`
  - **Tmux**: Use custom colors matching Tokyo Night palette
</Accordion>

<Accordion title="Nord">
  - **Neovim**: Install `shaunsingh/nord.nvim`
  - **Alacritty**: Use `nord.toml`
  - **Tmux**: Use `arcticicestudio/nord-tmux`
</Accordion>

## Testing Your Theme

<Steps>
  <Step title="Open a test file in Neovim">
    Open a code file to see syntax highlighting with your new theme.
  </Step>
  
  <Step title="Check all UI elements">
    Verify that status lines, splits, and popups look good in both Neovim and Tmux.
  </Step>
  
  <Step title="Test in different lighting">
    View your theme in both bright and dark environments to ensure readability.
  </Step>
</Steps>

<Warning>
Some themes may require terminal true color support. Ensure your terminal has `TERM=xterm-256color` or similar set correctly.
</Warning>
